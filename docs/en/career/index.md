# Career System

> Subtitle: From capability model to technical influence, build a quantifiable growth path rather than vague advice.

## Module Positioning

The growth of a frontend engineer is not automatic after writing code for a few more years. It requires actively building a capability model, planning a technical path, and accumulating technical influence. This module does not offer vague advice like "read more source code" or "solve more problems." Instead, it decomposes growth into quantifiable capability dimensions, executable growth ladders, and observable influence signals.

Whether you are a junior engineer just starting out or a mid-to-senior engineer aiming to break through to architect, you can find the corresponding advancement framework in this module.

## Core Topics

- **Capability model**: T-shaped vs π-shaped talent, quantified dimensions of technical depth and breadth, and capability radar charts.
- **Growth path**: Key leap points and bottlenecks from junior → intermediate → senior → staff → architect.
- **Technical influence**: Layered progression from code contributions, technical sharing, open-source participation, technical decision-making, to team enablement.
- **Career decisions**: Technical track vs management track, big tech vs startup, and timing judgment for job changes.
