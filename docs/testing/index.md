# 测试工程

> 副标题：从负载测试、端到端测试到单元测试，构建可验证的质量保障体系

## 模块定位

测试不是开发完成后的"收尾动作"，而是一套贯穿需求、设计、交付全生命周期的工程方法。本模块把测试拆解为"验证、隔离、回归"三组目标：验证系统在真实压力下的边界，隔离缺陷到具体层次，回归保障每次变更不引入新问题。

每类测试都配套工具链选型、场景设计与结果分析方法，避免"跑了测试但说不清结论"的伪覆盖。

## 核心主题

- **负载与性能测试**：测试计划、负载模型、JMeter 脚本参数化、TPS / 响应时间 / 错误率指标体系、瓶颈定位
- **端到端测试**：Playwright / Cypress 场景设计、断言策略、并行执行、稳定性治理
- **单元与集成测试**：Jest / Vitest 测试金字塔、Mock 策略、覆盖率边界、契约测试
- **监控与可观测性**：Grafana、APM、OCP 指标采集、95% 百分位响应时间分析
- **稳定性保障**：降级与熔断验证、容量规划、第三方依赖压力测试
